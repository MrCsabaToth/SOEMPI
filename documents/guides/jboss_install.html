Installation and setup of JBoss

  1. Get JBoss 4.2.3.GA-jdk6 version from here. Note: it's important to get the jdk6 version since you probably have version 6+ JDK.
  2. Unpack the source to the desired location.
    • Linux: /opt can be a good location. Unpack the JBoss to /opt/jboss-4.2.3-GA and make /opt/jboss symlink which points to that directory. Specify this /opt/jboss directory as your JBOSS_HOME, this way you can easily upgrade to a new JBoss version: you just have to unpack it to another directory and point the symlink to that new one.
      It's very important that JBoss and SOEMPI's configuration files should have sufficient privileges. If you are on your own desktop machine, give your user's ownership and group privileges to JBoss. (chown username -R $JBOSS_HOME, chgrp username -R $JBOSS_HOME). On a server machine create a jboss user and apply that user's privileges.
    • Windows: unpack JBoss whereever you want. It can be C:\jboss, or some C:\Users\username\Documents, or something.
    • Mac OS X: TODO.
  3. Set JBOSS_HOME environmental variable. On details of how to set an environment variable in general see the explanation at OPENEMPI_HOME.
  4. In case of memory issues add the -Xms128m -Xmx3g -XX:MaxPermSize=512m -XX:+CMSClassUnloadingEnabled -XX:+CMSPermGenSweepingEnabled JVM parameters to the JAVA_OPTS:
    • Linux: add it to JAVA_OPTS in the $JBOSS_HOME/bin/run.conf.
    • Windows: add it to JAVA_OPTS in the $JBOSS_HOME/bin/run.bat.
    The first two parameters should replace the existing ones in the file. With these parameters we give more memory for the JVM and try to force better GC.
  5. If you want to start JBoss using init.d script on Linux, use this file: JBoss init.d script, and refer it in the rc.d scripts.
